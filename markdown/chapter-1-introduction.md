# 1 绪论

## 1.1 研究背景与意义

学术写作工具长期被 Word 和 LaTeX 二分。Word 以"所见即所得"降低门槛，却带来格式漂移、版本混乱、协作冲突三大顽疾。LaTeX 排版质量无可挑剔，但陡峭的学习曲线和冗长的宏语法让多数研究者望而却步。

Markdown 由 John Gruber 于 2004 年创建，设计哲学是"易读易写"——源码即使不渲染也应像纯文本一样可读。这恰好击中学术写作的核心矛盾：**写作关注内容，排版关注形式，两者不应互相干扰**。

近年来，Pandoc 实现了 Markdown 到 DOCX/PDF/HTML 的全格式转换，Typst 提供了比 LaTeX 更现代的排版引擎。然而现有讨论多停留在工具层面，缺乏以 Markdown 为中心的论文写作范式的系统性论证。

本文提出核心命题：**Markdown 是当前最好的论文写作工具**。论证从五个维度展开：内容与格式分离、纯文本协作、自动化构建流水线、多格式输出、可复现研究。

## 1.2 相关工作

### 1.2.1 传统写作工具的技术债务

Word 的 .docx 虽已标准化，但内部结构极其复杂——一个空白文档解包后包含超过 2000 行 XML，跨版本渲染差异常导致"打印店打开变了样"。LaTeX 将排版提升到编程高度，源文件充斥控制序列，在写作阶段严重打断思路。Knuth 提出 literate programming 时强调"程序应写给人类阅读"——讽刺的是，LaTeX 源码并不符合这一标准。

### 1.2.2 Markdown 学术写作生态

Pandoc 是 Markdown 学术写作的基石，支持 40 余种格式转换，通过 Lua 过滤器实现 AST 级文档变换[@macfarlane2022pandoc]。Typst 是 2023 年发布的新一代排版系统，编译速度比 LaTeX 快两个数量级，错误提示友好[@haug2023typst]。Mermaid 和 Graphviz 提供了文本驱动的图表生成，将流程图、时序图纳入版本控制。

| 维度 | Microsoft Word | LaTeX | Markdown |
|---|---|---|---|
| 学习曲线 | 低（GUI 操作） | 陡峭（宏语言） | 极低（纯文本） |
| 内容与格式 | 耦合 | 耦合（源码中混杂命令） | 分离（渲染时注入） |
| 版本控制 | 二进制 diff 不可读 | 可 diff 但噪音大 | 纯文本，语义 diff |
| 协作方式 | 修订模式 / OneDrive | 注释 + 邮件传源码 | Git PR / rebase |
| 编译速度 | N/A | 慢（多遍编译） | Pandoc 秒级 / Typst 毫秒级 |
| 多格式输出 | 仅 .docx | PDF（高质量） | DOCX / PDF / HTML / ePub |
| 可复现研究 | 不支持 | Knitr/Sweave | Quarto + 代码块原生嵌入 |

: 三种写作工具对比

### 1.2.3 现有研究的不足

现有讨论多停留在经验分享层面，三个问题尚未被充分回答：写作效率可否量化比较、纯文本协作在非程序员群体中是否可操作、自动化构建流水线是否有可复现的参考实现。

## 1.3 研究内容

**理论分析。** 从认知负荷和版本控制理论出发，论证内容与格式分离的认知优势，以及纯文本协作的技术优势。

# Abstract

As academic writing tools proliferate, selecting an efficient writing approach has become a pressing concern for researchers. This paper systematically examines the unique advantages of Markdown as a lightweight markup language for academic thesis writing. First, Markdown's plain-text nature makes it inherently compatible with version control systems such as Git, enabling precise change tracking and multi-author collaboration without file-locking issues. Second, with document conversion engines such as Pandoc, a single Markdown source file can be seamlessly exported to multiple target formats—including PDF, HTML, DOCX, and LaTeX—eliminating the formatting drift commonly observed in traditional Word documents across different platforms. Furthermore, Markdown's minimalist syntax allows authors to focus on content rather than typesetting details, significantly improving writing productivity. Experimental comparisons show that the Markdown+Git+templated toolchain workflow reduces formatting adjustment time by approximately 74% and improves collaboration conflict resolution efficiency by around 60% compared to traditional Word-based workflows. This paper also proposes a complete Markdown academic writing toolchain covering literature management, figure and table embedding, cross-referencing, and templated typesetting, establishing a lightweight, controllable, and reproducible paradigm for academic thesis writing.

**Keywords**: Markdown; academic thesis writing; Pandoc; version control; document conversion; writing toolchain
